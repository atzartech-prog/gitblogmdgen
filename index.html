Langkah 1

Membuat Repositori Baru di GitHub

GitHub Pages memerlukan repositori publik khusus untuk menghosting file situs web Anda secara gratis.

Langkah Detail:

  1. Buka situs GitHub.com dan masuk menggunakan akun Anda.
  2. Klik tombol "New" atau klik ikon plus (+) di kanan atas lalu pilih "New repository".
  3. Pada kolom Repository name, masukkan nama dengan format persis seperti ini:
    username.github.io
    (Ganti kata username dengan nama akun GitHub Anda secara persis. Jika username Anda "budi", nama repositorinya wajib "budi.github.io").
  4. Pastikan visibilitas diatur ke **Public**. GitHub Pages tidak dapat diaktifkan secara gratis pada repositori privat (Private).
  5. Centang opsi **Add a README file** untuk mempermudah inisialisasi awal.
  6. Klik tombol hijau **"Create repository"** di bagian paling bawah.
Catatan Sangat Penting:

Ekstensi ".github.io" di belakang username wajib ditulis dengan benar agar server GitHub mengenali bahwa repositori ini akan di-publish sebagai situs web.

Langkah 2

Membuat File Konfigurasi Jekyll (_config.yml)

Jekyll adalah parser statis bawaan GitHub Pages. Untuk mengaktifkan tema Jekyll, kita memerlukan file konfigurasi bernama _config.yml.

Langkah Detail:

  1. Di dalam repositori username.github.io Anda yang baru dibuat, klik tombol "Add file" lalu pilih "Create new file".
  2. Beri nama file tersebut: _config.yml
  3. Salin kode konfigurasi di bawah ini dan tempelkan ke dalam editor teks GitHub:
YAML 
title: Blog Pribadi Saya
description: Catatan harian mengenai pemrograman dan tips teknologi
theme: jekyll-theme-minimal

Pilihan Tema Resmi Lainnya:

Anda dapat mengganti nilai pada baris theme: dengan salah satu tema Jekyll yang didukung resmi oleh GitHub Pages di bawah ini:

  • jekyll-theme-minimal (Tema minimalis bersih)
  • jekyll-theme-cayman (Tema dengan gradasi warna biru-hijau tebal)
  • jekyll-theme-architect (Tema bernuansa teknikal modern)
  • jekyll-theme-slate (Tema bernuansa dark mode abu-abu)
  • jekyll-theme-merlot (Tema dengan layout sidebar merah tua)
  1. Klik tombol hijau "Commit changes..." di kanan atas halaman untuk menyimpan file.
Langkah 3

Membuat Halaman Utama (index.md)

File index.md akan dibaca oleh Jekyll sebagai halaman muka/halaman beranda blog Anda saat pertama kali diakses.

Langkah Detail:

  1. Kembali ke root repositori Anda, klik "Add file" lalu pilih "Create new file".
  2. Beri nama file tersebut: index.md
  3. Salin kode template beranda di bawah ini dan tempelkan ke dalam editor teks:
Markdown + Front Matter 
---
layout: default
title: Beranda
---

# Selamat Datang di Blog Saya!

Halo pembaca, ini adalah situs web statis saya yang dibuat secara langsung dari file **Markdown** dan di-host di **GitHub Pages** menggunakan engine **Jekyll**.

### Daftar Postingan Terbaru
Berikut adalah daftar catatan harian saya:

{% for post in site.posts %}
* [{{ post.title }}]({{ post.url }}) - {{ post.date | date: "%d %B %Y" }}
{% endfor %}
Info Syntax {% ... %}:

Bagian bawah kode di atas menggunakan bahasa template Liquid bawaan Jekyll. Kode tersebut secara otomatis akan memindai folder _posts dan menampilkan daftar artikel Anda secara dinamis.

  1. Klik tombol hijau "Commit changes..." untuk menyimpan file.
Langkah 4

Menulis Postingan Artikel (_posts/...)

Semua tulisan blog atau postingan harus disimpan di dalam folder bernama _posts, dan penamaan file wajib mengikuti format tertentu agar bisa terindeks oleh mesin Jekyll.

Aturan Penulisan Postingan:

  1. Nama Folder: Di root direktori repositori Anda, buat folder baru bernama `_posts`.
  2. Penamaan File Artikel: Wajib menggunakan format format tanggal dan slug judul seperti berikut:
    YYYY-MM-DD-judul-tulisan.md
    Contoh: 2026-06-20-belajar-github-pages.md
  3. Front Matter (Header Metadata): Di bagian paling atas file markdown Anda, wajib menyertakan Front Matter yang diapit oleh tanda tiga strip (`---`).

Contoh Postingan Lengkap:

Buat file baru di dalam folder _posts dengan nama 2026-06-20-artikel-pertama.md dan masukkan isi berikut:

Markdown + Front Matter 
---
layout: post
title: "Artikel Pertama: Cara Mulai Menulis"
date: 2026-06-20
categories: blog tutorial
tags: [github, jekyll]
---

# Pendahuluan

Selamat datang di postingan blog pertama saya! Hari ini saya belajar cara mempublikasikan artikel di blog GitHub menggunakan Jekyll.

### Mengapa Memakai Markdown?
* Sangat ringan dan mudah dibaca.
* Tidak membutuhkan tag HTML yang rumit.
* Terintegrasi langsung dengan editor GitBlogMD.

Sekian tulisan singkat dari saya, selamat mencoba!
  1. Klik tombol hijau "Commit changes..." untuk menyimpan postingan pertama Anda.
Langkah 5

Mengaktifkan & Membuka Blog Anda

Setelah seluruh file konfigurasi, halaman utama, dan artikel pertama Anda buat, langkah terakhir adalah mengaktifkan fitur GitHub Pages dan mempublikasikannya ke internet.

Langkah Aktivasi:

  1. Buka halaman utama repositori Anda di GitHub.
  2. Klik tab "Settings" yang berada di bagian atas menu navigasi repositori (sebelah tab "Insights").
  3. Pada panel navigasi sebelah kiri, klik menu "Pages" di bawah bagian Code and automation.
  4. Di bagian Build and deployment:
    • Pastikan drop-down Source memilih opsi "Deploy from a branch".
    • Di bagian Branch, ubah pilihan dari None menjadi `main` (atau master) lalu pilih folder `/ (root)`.
    • Klik tombol "Save" di sebelahnya.
  5. Tunggu proses kompilasi berjalan sekitar 1 hingga 2 menit. Anda bisa me-refresh halaman settings tersebut.
  6. Setelah selesai, GitHub akan memunculkan banner notifikasi sukses berwarna hijau di atas halaman Pages Anda yang berisi alamat URL langsung blog Anda:
    Your site is live at: https://username.github.io/
  7. Klik tautan tersebut untuk melihat blog Jekyll kustom Anda yang cantik!
Selesai!

Selamat! Sekarang situs blog Anda telah aktif di seluruh dunia secara gratis selamanya.

Alur Kerja Cepat

Menggunakan GitBlogMD Untuk Menulis Harian

Aplikasi GitBlogMD ini dirancang untuk memudahkan Anda menulis secara offline tanpa pusing dengan struktur file Jekyll.

Cara Penggunaan Sehari-hari:

  1. Buka menu "Tulis Postingan" pada aplikasi ini.
  2. Tulis judul artikel baru Anda di kolom **Judul Artikel**. Aplikasi secara otomatis akan mem-format nama file keluar agar pas dengan konvensi Jekyll.
  3. Tulis konten postingan Anda di area teks editor menggunakan Markdown. Gunakan **Live Preview** untuk melihat langsung visual akhirnya.
  4. Jika belum selesai, klik tombol **"Simpan Database"** untuk menyimpannya sebagai draf lokal di browser Anda. Draf ini bisa dibuka kembali kapan saja di menu **"Daftar Postingan"**.
  5. Jika sudah siap dipublikasikan:
    • Klik tombol **"Download .md"** untuk mengunduh file postingan secara langsung ke perangkat Android/Komputer Anda.
    • Buka repositori username.github.io Anda di GitHub.
    • Masuk ke folder `_posts`.
    • Klik "Add file" -> "Upload files".
    • Pilih file .md yang baru saja Anda unduh tadi, lalu lakukan **Commit changes**.
  6. Selesai! GitHub Pages akan me-build artikel baru tersebut secara otomatis dalam beberapa detik.
